## Affected Pages/Routes
- POST /webhook/leads — Sales agent lead intake trigger; test qualification, HITL escalation, memory write
- GET /dashboard — Operator dashboard; test pending_review panel, ops_alert panel
- Event bus subscribe handler — test async error routing to dead-letter

## Key Interactions to Verify

### HITL Threshold (P1 — test before any agent touches production data)
- outbound email to NEW contact → escalates
- outbound email to contact IN existing thread → does NOT escalate
- spend > $0 → escalates
- spend = $0 → does NOT escalate
- contract action → escalates
- internal report → does NOT escalate
- CRM update → does NOT escalate
- draft creation → does NOT escalate
- HITL timeout → task appears in ops/pending_review/{task_id}
- HITL approved → task executes
- HITL rejected → outcome:rejected logged

### BaseAgent Lifecycle
- memory_store() completes successfully before publishEvent() fires
- subscriber reads committed data (not stale) when processing event
- handler rejection routes to ops_alert dead-letter
- 5-step lifecycle executes in correct order

### Event Bus
- publish() → subscriber handler called
- filter by source_agent works
- retry 3x with 30s backoff on delivery failure
- after 3 failures → ops_alert key written
- handler Promise rejection → caught, routed to dead-letter

### Concurrency Cap
- ≤5 parallel HITL escalations → all proceed immediately
- 6th escalation → queued until a slot frees
- operator dashboard shows queue depth

### Memory Key Schema
- hierarchical key `{agent}/{entity_type}/{entity_id}` constructed correctly
- CRM-assigned ID is used as entity_id (not surrogate numeric ID)
- no key collision between agents writing to different namespaces

### Metrics
- every task logs {agent, task_id, trigger_source, escalated, completion_time_s, outcome, retrieval_time_ms}
- outcome values: success, failed, escalated-approved, escalated-rejected, timeout

## Edge Cases
- 0 leads webhook → Sales agent handles gracefully
- Malformed webhook payload → validation error, not agent execution
- AgentDB unreachable → agent pauses, logs error, does not execute
- memory_store() failure → publishEvent() not called
- HITL notification channel not configured → agents do not start background tasks

## Critical Paths
1. [E2E] Lead arrives via webhook → Sales qualifies → HITL fires for outbound → operator approves → Marketing receives event → campaign draft written to memory
2. [E2E] HITL timeout at 3am → task moves to pending_review → operator sees it in dashboard next morning
3. [E2E] Event bus delivery failure → 3 retries → ops_alert → dashboard panel shows alert
